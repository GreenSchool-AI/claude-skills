# Contributing

Thanks for thinking about contributing! These skills are practical tools — feedback from
people who actually use them makes them better.

## What we want

**Bug reports.** "I asked the resume skill to do X and it gave me Y, which is wrong." The
more specific, the better. Include the input you used and what came back.

**Real-world examples.** Did the skill help you land an interview? Win a client? Fix an
embarrassing email before sending it? Share the (anonymised) story — it helps other
users see what's possible and helps us write better examples.

**Edge cases we missed.** "I tried this with a healthcare resume and the AU-specific
formatting was off because of AHPRA registration requirements." We probably haven't
covered every industry. If you know one of them well, your input is gold.

**Translations and cultural notes.** Most of our users have English as a second language.
If a skill output sounds awkward in a particular cultural context, tell us.

## What we're cautious about

**Adding more skills.** Each skill in this repo is maintained by the Green School AI team
because we use them ourselves and stand behind them. We're slow to add new skills from
PRs because we want to keep quality consistent. If you've built something great — open
an Issue or Discussion first to talk it through.

**Major rewrites.** A SKILL.md is the result of a lot of iteration. Don't rewrite the
whole thing in one PR; suggest a specific change, ideally with a before/after example
showing why it's better.

## How

### Bug or suggestion

Open an [Issue](https://github.com/GreenSchool-AI/claude-skills/issues). Templates will
guide you through what to include.

### Discussion

Open a [Discussion](https://github.com/GreenSchool-AI/claude-skills/discussions) for
broader topics — feature ideas, sharing your prompting tricks, asking how to use a skill
in your specific situation.

### Pull request

1. Fork the repo.
2. Make your change in a focused branch (one fix or one improvement per PR).
3. If you're changing a SKILL.md, include a before/after example showing the
   improvement.
4. Open the PR with a clear description of what changed and why.

We review on weekends. If we don't reply within a week, ping the PR — we sometimes miss
notifications.

## Code of conduct

Be kind. Disagreements are fine; personal attacks aren't. We follow the
[Contributor Covenant](https://www.contributor-covenant.org/).
